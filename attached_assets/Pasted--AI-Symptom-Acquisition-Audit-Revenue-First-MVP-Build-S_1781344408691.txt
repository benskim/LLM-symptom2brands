# AI Symptom Acquisition Audit™

## Revenue-First MVP Build Spec v3 (Gemini Edition)

### SYSTEM ROLE

You are a senior software engineer.

Your job is NOT to build a perfect GEO platform.

Your job is to build the smallest possible system that can generate a sellable audit report for a Shopify gut health brand.

The founder is a solo developer.

The founder needs paying customers, not infrastructure.

1. Why competitors are recommended
2. Why their brand is missing
3. What actions they should take

# MODULE A

Visibility Sampling Engine

Purpose:

Measure recommendation frequency.

Input:

Brand name

Prompt set

Example prompts:

* best probiotic for bloating
* supplements for constipation
* digestive support for kids
* gut health products for families
* best probiotics for IBS

Process:

For each prompt:

1. Create fresh Gemini request
2. No memory
3. No conversation history
4. Run 5 samples
5. Store raw responses

Execution:

Sequential only.

No async.

Output:

{
"visibility_score": 20.0,
"mentions": 5,
"samples": 25
}

Formula:

mentions / samples * 100

---

# MODULE B

Competitor Aggregation Engine

Purpose:

Identify which brands appear most frequently.

Process:

Extract brand names from all responses.

Generate frequency table.

Output:

{
"Seed": 12,
"Bioma": 9,
"Pendulum": 5,
"Begin Health": 2
}

Do not build:

* knowledge graphs
* entity resolution systems
* embeddings

Simple frequency counting only.

---

# MODULE C

Citation Surface Engine

Purpose:

Identify which sources Gemini references.

Extract:

* URLs
* domains
* cited publications

Output:

{
"top_domains": [
"healthline.com",
"reddit.com",
"webmd.com"
]
}

Calculate:

citation_share

Formula:

brand_domain_mentions /
total_citation_mentions

Fallback:

If structured citations unavailable:

Extract URLs via regex.

---

# MODULE D

Grounding Classification

Supported:

STATIC
WEB

Only these two.

Do not implement:

* shopping feeds
* merchant center
* ecommerce feed diagnostics

Output:

{
"static_mentions": 8,
"web_mentions": 17
}

---

# MODULE E

Evidence Gap Analyzer

This is the most important module.

Input:

* visibility score
* competitors
* citation data
* grounding data

Model:

Gemini

Temperature:

0

Rules:

Never invent facts.

Allowed:

"Possible gap"

"Likely gap"

"Observed difference"

Forbidden:

"Root cause"

"Guaranteed explanation"

"Definitive reason"

Output:

{
"gap_type": "authority_signals",
"confidence": "medium",
"evidence": [
"Healthline cited 8 times",
"Brand cited 0 times"
]
}
